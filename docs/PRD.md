# 제품 요구사항 정의서 (PRD)

**프로젝트명:** 최적가 항공권 모니터링 AI 에이전트 (MVP)
**문서 버전:** 1.0
**작성일자:** 2026년 4월 15일
**작성자:** 권순현 (Product Owner)
**수신:** Antigravity 개발팀

---

## 1. 프로젝트 개요 (Project Overview)

본 프로젝트는 사용자가 지정한 목적지의 항공권 가격을 매일 모니터링하고, 과거 데이터 기반의 역대 최저가와 비교하여 일정 비율(최저가 + 20%) 이내로 가격이 하락했을 때 자동으로 알림을 발송하는 **'AI 에이전트 기반 자동화 시스템'**을 구축하는 것을 목표로 합니다. 

초기 MVP(Minimum Viable Product) 단계에서는 복잡도를 낮추고 핵심 파이프라인 검증에 집중하기 위해 **'인천(ICN) - 다낭(DAD)'** 노선을 단일 타겟으로 설정합니다.

## 2. 시스템 아키텍처 및 개발 환경 (Architecture & Environment)

본 시스템은 클라우드 LLM API 비용을 최소화하고 보안을 강화하기 위해 **로컬 LLM** 환경에서 구동되도록 설계되어야 합니다.

* **에이전트 프레임워크:** Google ADK (Agent Development Kit) 기반 Python 애플리케이션
* **LLM 엔진 (로컬):** Ollama
* **타겟 LLM 모델:** `Llama-3.1` (8B, 4-bit 양자화) - *1순위* / `Phi-3-mini` (3.8B) - *2순위*
* **데이터 소스:** SerpApi (Google Flights API)
* **운영 하드웨어 (Target H/W):**
    * CPU: Intel Core i7-1165G7 (AVX-512 VNNI 지원 환경)
    * RAM: 16GB (OS 점유율 고려, LLM 가용 램 8~10GB 제한)
    * GPU: 외장 그래픽카드 없음 (CPU Inference 전용)

## 3. 핵심 아키텍처 패턴: Sub-agent 위임 (Handoff)

에이전트는 역할을 분리하여 안정성과 효율성을 극대화합니다.

### 3.1. Main Agent (오케스트레이터)
* **역할:** 사용자 의도 파악, 전체 워크플로우 지휘, 알림 발송 여부 최종 판단.
* **동작 방식:** 로컬 LLM을 사용하여 Sub-Agent #1이 반환한 JSON 결과 데이터를 분석하고, 설정된 타겟 가격 기준을 충족할 경우 사용자에게 알림을 발송합니다.

### 3.2. Sub-Agent #1 (데이터 수집 및 분석 전담)
* **역할:** 구글 플라이트 API(SerpApi)를 호출하여 실질적인 가격 데이터를 수집하고 수학적 계산(최저가 기준점 산출)을 수행합니다.
* **동작 방식:** LLM의 환각(Hallucination)을 방지하기 위해 API 호출과 가격 비교 계산은 모두 파이썬 Tool 내부에 하드코딩된 로직으로 수행되며, 에이전트는 해당 Tool을 실행하기만 합니다.

## 4. 기능 요구사항 (Functional Requirements)

### FR-1. 최적화된 API 호출 (전략 B 알고리즘 적용)
단일 목적지에 대한 탐색 시, API 비용 절감과 로컬 환경의 타임아웃 방지를 위해 **'편도 선행 탐색 후 왕복 결합(Two-Phase)'** 방식을 의무 적용합니다.
1.  **Phase 1 (출발일 추출):** 검색 시작일(오늘)로부터 30일간의 '편도' 항공권 가격을 조회하여 가장 저렴한 출발일 Top 3를 추출합니다.
2.  **Phase 2 (왕복 탐색):** 추출된 Top 3 출발일에 대해서만 4일~14일 체류하는 왕복 항공권을 조회합니다.
    * 탐색 최적화를 위해 체류 기간은 **짝수일(4, 6, 8, 10, 12, 14일)** 단위로만 검색합니다. (호출 횟수 제한)

### FR-2. 역대 최저가 및 타겟 가격 산출
* SerpApi의 `price_insights` 데이터를 파싱하여 해당 노선의 과거 데이터 기반 '역대 최저가'를 확인합니다.
* Python Tool 내부 로직을 통해 **[발견된 최저가 * 1.2]** 수식을 적용하여 '알림 타겟 가격(+20%)'을 자동 산출하고, 이를 Main Agent에게 JSON 형태로 반환해야 합니다.
* **가격 기준:** 세금 및 유류할증료가 모두 포함된 '최종 총액(KRW)'을 기준으로 합니다.

### FR-3. 로컬 LLM 연동 설정
* Ollama 서버(`http://localhost:11434/v1`)를 바라보도록 OpenAI 클라이언트 베이스 URL을 수정하여 ADK와 통합합니다.

### FR-4. 매일 자동화 실행
* Windows 작업 스케줄러 또는 GitHub Actions/Cron 등을 활용하여, 개발된 Python 스크립트가 매일 지정된 시간에 1회 자동 실행되도록 구성해야 합니다.

### FR-5. 알림 전송 기능은 지금은 구현하지 않는다.
* 목표 가격 도달 시 사용자에게 즉각적인 알림을 발송하는 Tool을 연동합니다.
* **수단:** Telegram Bot API (MVP 우선 적용) 또는 Slack/Email 연동.
* **알림 내용:** 최적 출발/귀국 날짜, 여행 기간, 항공사, 세금 포함 최종 가격, 예약 링크.

## 5. 비기능 요구사항 (Non-Functional Requirements)

* **API Rate Limit 방어:** 다수 호출이 발생하는 Phase 2 검색 시, 봇 차단(`429 Error`)을 막기 위해 각 API 호출 사이에 의도적인 지연 시간(Sleep/Jitter, 최소 0.5초~1초)을 구현해야 합니다.
* **메모리 최적화:** RAM 16GB (가용 8GB) 환경이므로, LLM 구동 시 메모리 누수가 발생하지 않도록 컨텍스트 윈도우(Context Window)를 최소한으로 관리해야 합니다.
* **예외 처리:** 특정 날짜에 항공권이 없거나 API 응답 지연 시 스크립트가 중단되지 않고, 에이전트에게 자연어로 에러 상태(`status: error`)를 반환하도록 설계해야 합니다.
